UTAHCARSAFETY.com

Three of my classmates and I built UtahCarSafety.com during the INTEX 2022 competition. We place 4th out of over 50 teams! We spent around 70 hours each on the project.

The problem that we were faced with is the high percentage of fatal car crashes in Utah. Our solution was to create UtahCarSafety.com to raise awareness through touch points between drivers and regulators. However, the site is no longer publically hosted because of thee high monthly AWS cost.

UtahCarSafety.com Functionalities:

  1. The application features an extensive database with thousands car crashs throughout the state of Utah. We added the ability to filter through the crash by time, severity, city, and county. Selected filters are dynamically shown on the page. There is also an “Advanced Filter” option that allows the user to filter by any variable.
  2. Add/Edit/Delete to the database is only possible with Admin access.
  3. Applied ML Model: The Crash Details page includes a “What If” functionality that shows how much the severity score of the accident would decrease if an important factor of the crash wasn’t included. There is also a Crash Scenarios page that allows the user to find what the crash severity score would be for a hypothetical crash.
  4. Multi-Factor Authentication through a user's google account.

OPIOIDSAFTEY.com

The Opioid Safety application was my first big project. Three of my classmates and I spent a full week (~70 Hours each) working on this project in the INTEX 2021 competition.

The web application displays data on prescribers and prescription drugs, allows the user to add, edit, delete, and search prescribers and drugs, and provides various analyses on prescribed opioids. This website and all related data is meant to be used by government agencies to fight the opioid crisis, as well as the prescribers themselves to be more educated on the dangers of opioids.

Starting with the most basic feature, OpioidSaftey.com contains detailed information about the dangers of opioids and Substance Use Disorder (SUD). The purpose of the site is to educate readers about how quick opioid SUD can develop, and about the severity of the current opioid crisis. We used datasets which included details about drugs, prescribers, and states. Using this data, we created a predictor and recommender in a machine learning environment. We implemented both in our website so that stakeholders will be able to input their information and receive a prediction or recommendation based on specific criteria. The predictor helps government agencies find prescribers who are more at-risk to over-prescribe opioids, and the recommender will help prescribers in finding non-opioid alternatives.
